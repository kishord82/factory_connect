1. Phase 1 Product Definition — The One-Sentence Version

"Keep your ERP. FactoryConnect handles buyer-ready documents, onboarding, and export-compliance workflows."

Phase 1 is NOT a universal integration platform. It is an India-first export-enablement SaaS for Tally/Zoho-based exporters needing buyer integration and CBAM-driven export compliance.

1.1 What Phase 1 Sells

CapabilityDescriptionWhy It Matters
CBAM carbon reporting Automated data extraction from ERP → EU-compliant quarterly CBAM reports Mandatory since Jan 2026. Factories spending Rs 2-5 lakh/quarter on consultants. Immediate urgent pain.
Buyer document generation PO intake → sales order → invoice → shipment/ASN → buyer-format output Replaces manual re-keying (2-3 hrs/order). Eliminates buyer chargebacks for format errors.
Tally/Zoho integration Bridge agent (Tally) or API connector (Zoho) reads/writes ERP data "Keep your ERP" is the key message. No ERP migration. Just plug in.
Buyer template library Pre-built templates for 5-10 buyer document specs (EDI X12, cXML, Ariba) Factory doesn't need to understand EDI. Templates handle translation.
Managed transport AS2/VAN delivery via TrueCommerce xChange partnership No factory needs to manage AS2 certificates. FC handles transport.

1.2 The Narrow Workflow Corridor

Phase 1 supports ONE transaction flow end-to-end:

PO Intake → Sales Order Sync/Create in ERP → Invoice Output → Shipment/Dispatch Output → Buyer-Format Delivery

Plus: CBAM compliance overlay — emissions data extraction from shipment/production records → quarterly EU report generation.

That's it. No inventory forecasting, no analytics dashboards, no AI auto-healing, no WhatsApp order intake, no multi-country expansion. Those are Phase 2+.

2. Hard Scope Boundaries — What's IN vs OUT

✅ IN Phase 1 — Build This

  • Source ERPs: Tally Prime (JSON-first, XML fallback) + Zoho Books/Inventory
  • Core entities: Order, Invoice, Shipment, Party, Product, Inventory
  • Workflow: PO → SO → Invoice → Shipment (one corridor)
  • Compliance: CBAM carbon reporting module
  • Buyer delivery: Template-driven document generation, managed AS2/VAN
  • Buyer templates: 5-10 buyer specs, manually curated
  • Mapping: Configuration UI with AI-suggested, human-approved mappings
  • Operations: Audit trail, retry queues, DLQ, status dashboard, exception console
  • Notifications: WhatsApp + email for alerts and onboarding ONLY
  • Security: Multi-tenant with RLS, parameterized queries, PII redaction, FLE for sensitive fields
  • Onboarding: Ops-heavy, human-assisted with product checklists
  • Billing: Stub for Phase 1 (manual invoicing OK for first 50 customers)
  • Sandbox/demo: Prospects can see the product working with sample data

❌ NOT in Phase 1 — Explicitly Deferred

  • ERPs: SAP B1, Marg, Busy, ERPNext — wait for paying demand
  • WhatsApp-first product: No-ERP factories using WhatsApp as primary channel — this is a SEPARATE product motion, not Phase 1
  • Full AI auto-healing: AI suggests mappings, but no autonomous correction in Phase 1
  • In-house AS2 transport: Use managed VAN only. Build own AS2 in Phase 2 when volume justifies
  • Buyer self-serve onboarding: Factory onboarding is human-assisted in Phase 1
  • Multi-country expansion: India only. No EDIFACT/EU-specific buyer formats beyond CBAM
  • Advanced analytics: No AI forecasting, no demand planning, no BI dashboards
  • Full field-sales org: Founder-led sales + 1 hire max in Phase 1
  • Buyer network partnerships at scale: 1-2 buyer design partners max
  • Direct buyer portal / white-labeling: Phase 2
  • Deep workflow engine: No saga orchestrator with 15 states. Simple state machine with manual exception handling.
Scope Discipline Rule: If any feature request or "good idea" doesn't directly serve the narrow workflow corridor (PO → SO → Invoice → Shipment + CBAM), it goes on the Phase 2 backlog. No exceptions. This is the single most important discipline for a solo/small-team startup. Every feature you DON'T build buys you time to get the ones you DO build right.

5. 120-Day Execution Plan

Assumption: This plan is calibrated for a realistic trajectory: solo founder in Month 1, growing to 3-4 people by Month 4. Not ChatGPT's 5-person team from Day 1 (that's ideal but unlikely without funding). Adjusted deliverables accordingly.

Days 1-15: Validation Sprint (BEFORE any code)

Goal: Confirm factories will pay before building anything.

  • Cold-message 50 steel/metal exporters on LinkedIn + WhatsApp
  • Complete 10 validation interviews: "How do you handle CBAM? What do you pay? Would you pay Rs 5K/month?"
  • Talk to 3-5 CBAM consultants: "What do clients struggle with? What would an automated tool need?"
  • Create landing page with pricing + waitlist. Run Rs 10K Google Ads targeting "CBAM compliance India"
  • GO/NO-GO gate: If 7+ interviews show strong interest AND 15+ waitlist signups → PROCEED. If not → REASSESS.

Days 15-30: Foundation + CBAM Module Start

Goal: Core infrastructure + CBAM module skeleton.

  • Finalize wedge: CBAM-first exporters on Tally
  • Set up monorepo, Docker Compose, CI pipeline
  • Design tenant model, core DB schema (PostgreSQL + RLS), audit/event model
  • Build auth (Keycloak integration) + tenant context middleware
  • Start Tally bridge agent MVP (JSON API for TallyPrime 7.0+)
  • Start CBAM emissions data collection module
  • Start canonical order/invoice/shipment pipeline
  • Hire decision: Post for 1 backend engineer (Go). Needed by Day 30-45.

Days 31-60: Core Product Build

Goal: Tally connector working + CBAM module functional + mapping UI.

  • Complete Tally bridge agent (JSON-first + XML fallback for pre-7.0)
  • Build canonical model service with validation
  • Build mapping configuration UI (AI-suggested, human-approved)
  • Build CBAM calculation engine + EU report generation
  • Add pre-validation engine
  • Add idempotency + retry queues + DLQ (BullMQ)
  • Build manual mapping review UI
  • Create 3 buyer templates (maximum)
  • Start outreach to first 3-5 design partner factories
  • Set up sandbox/demo environment with sample steel exporter data

Days 61-90: Design Partner Pilots

Goal: 3-5 factories in controlled pilot. First real data flowing.

  • Onboard 3-5 design partners (steel exporters, free or discounted)
  • Complete document status dashboard + exception handling console
  • Add audit log with hash-chain
  • Add Zoho connector if demand exists from pilots
  • Add buyer-specific template overrides based on pilot feedback
  • Build monitoring + alerting (Grafana + Prometheus)
  • First CBAM quarterly reports generated for pilot factories
  • Run first CBAM webinar / outreach cycle
  • Instrument: onboarding time, mapping accuracy, message failure rates
  • Hire decision: Need 1 EDI/integration specialist or strong contractor by now.

Days 91-120: First Revenue

Goal: Convert pilots to paid. 5-10 paying customers.

  • Convert design partners to paid subscriptions (Rs 5K-15K/month)
  • Add 5 more factories from CBAM outreach pipeline
  • Harden: retry logic, error handling, Tally bridge stability
  • Add billing stub and pricing experiments
  • Package onboarding as standardized service
  • Write support runbook for top 20 issues
  • Publish first case study from pilot success
  • Start Tally dealer partner recruitment in 2 export-hub cities
  • Decide: does CBAM remain primary wedge or become upsell layer?
  • TARGET: 5-10 paying customers, Rs 50K-1.5L MRR

Lane A: CBAM Urgency (Fastest to Revenue)

Target: Steel/aluminium/chemical exporters to EU

Message: "CBAM is mandatory since Jan 2026. You're paying consultants Rs 2-5 lakh/quarter. We automate it for Rs 5K/month."

Target: Tally implementation partners in export-hub cities

9.1 Proof Standard — When Is Product-Market Fit Proven?

Don't ask "do customers like this?" Ask these specific questions:

  • Did a factory go live with one buyer? (Technical proof)
  • Did messages flow without buyer rejection? (Quality proof)
  • Did the factory reduce manual work measurably? (Value proof)
  • Did the buyer accept the format without custom firefighting? (Scalability proof)
  • Did the compliance output unblock shipments or reduce cost? (ROI proof)
  • Did the factory agree to pay after the pilot? (Willingness-to-pay proof)

If all 6 are YES for 5+ customers → product-market fit confirmed. Scale.

Final Word

Both Claude's Market Viability Analysis and ChatGPT's strategic review converge on the same conclusion: FactoryConnect is a viable product in a real market, but only if you ship a narrow wedge first.

The Phase 1 product is: "CBAM-led export-enablement SaaS for Tally-based Indian exporters, with buyer-document integration for a small set of buyer formats and heavy human-assisted onboarding."
