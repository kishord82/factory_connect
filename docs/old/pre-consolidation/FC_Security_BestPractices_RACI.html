2. RACI Legend

R = Responsible (does the work) A = Accountable (final authority, signs off) C = Consulted (provides input before action) I = Informed (notified after action)
On-Prem = Factory with on-premises ERP only Cloud = Factory using cloud ERP (Zoho, ERPNext) Hybrid = Mix of on-prem + cloud systems

4. Environment-Specific Security Playbooks

4.1 On-Premises Factory (Tally Prime / Busy / Marg)

LayerBest PracticeOwnerTooling
ConnectivityInstall FactoryConnect Bridge Agent on factory LAN. Outbound-only HTTPS to FC cloud. No inbound ports opened.Cust-IT (install) + FC-Ops (config)FC Bridge Agent, WireGuard VPN
AuthenticationBridge agent uses rotating JWT tokens (auto-renewed every 24h). Token stored in OS keychain, not plaintext.FC-Eng (build) + Cust-IT (deploy)Keycloak, OS Keychain API
Data in TransitAll Tally XML data encrypted with TLS 1.3 in tunnel. No factory data touches public internet unencrypted.FC-OpsTLS 1.3, mTLS
Local DataBridge agent caches failed syncs locally (encrypted). Auto-retries when connection restored. Max 48h local cache.FC-EngSQLite (encrypted), AES-256
UpdatesBridge agent auto-updates via signed packages. Factory IT can defer updates up to 7 days. No silent changes.FC-Eng (package) + Cust-IT (approve)Code signing, semver
MonitoringAgent heartbeat every 60s. Alert FC-Ops + Cust-IT if agent goes offline for >5 min.FC-OpsPrometheus, Novu alerts
Key principle for on-prem: The factory never opens inbound ports. The bridge agent always initiates outbound connections. This eliminates 90% of attack surface for factories with basic firewalls.

4.2 Cloud Factory (Zoho Books / ERPNext / SAP B1 Cloud)

LayerBest PracticeOwnerTooling
ConnectivityOAuth 2.0 / API key authentication with ERP cloud APIs. No VPN needed. Webhook verification (HMAC signatures).FC-Eng + Cust-IT (API key provisioning)OAuth 2.0, HMAC-SHA256
AuthorizationRequest minimum scopes from ERP API. Read-only for sync; write-only for order confirmation. Principle of least privilege.FC-Eng (implement) + FC-Sec (review)OAuth scopes, Zoho API Console
Webhook SecurityAll inbound webhooks verified via shared secret HMAC. IP allowlisting where supported (Zoho, SAP).FC-EngHMAC, IP allowlist
Token StorageOAuth tokens encrypted in Vault/KMS. Never in application config or database plaintext.FC-Eng + FC-OpsHashiCorp Vault, GCP KMS
Rate LimitsRespect ERP API rate limits. Implement exponential backoff. Queue overflow goes to DLQ for retry.FC-EngBullMQ, Redis rate limiter

4.3 Hybrid Factory (Tally on-prem + Zoho Inventory cloud)

LayerBest PracticeOwnerTooling
ArchitectureBridge agent handles Tally (on-prem). Cloud connector handles Zoho (cloud). Both feed canonical model. No cross-contamination of credentials.FC-EngSeparate connector instances
Data ReconciliationCanonical model validates data from both sources. Conflict resolution rules defined per tenant. Audit trail for all merges.FC-Eng + Cust-ITJSONata, audit log
Credential IsolationTally bridge token and Zoho OAuth token stored in separate Vault paths. Compromising one doesn't expose the other.FC-Eng + FC-OpsVault namespaces
FailoverIf one source goes down, the other continues. No single point of failure. Manual reconciliation UI for conflicts.FC-Eng + FC-OpsBullMQ DLQ, conflict UI
Hybrid is the most complex scenario. Recommend: onboard factories as single-source first (Tally OR Zoho), then add second source after 30 days of stable operation.
